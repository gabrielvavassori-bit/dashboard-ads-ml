O QUE VOCÊ PRECISA FAZER (3 PASSOS - ~30 MINUTOS)
================================================================================

PASSO 1: GitHub (5 minutos)
  1. Acesse https://github.com/new
  2. Crie repositório "dashboard-ads-ml" (Public)
  3. Execute no terminal:
     cd /home/ubuntu/dashboard-ads-ml
     git remote add origin https://github.com/SEU_USUARIO/dashboard-ads-ml.git
     git branch -M main
     git push -u origin main

PASSO 2: Render.com (10 minutos)
  1. Acesse https://render.com
  2. Clique "New +" → "Blueprint"
  3. Selecione repositório "dashboard-ads-ml"
  4. Preencha variáveis de ambiente:
     - APP_PUBLIC_URL: https://dashboard-ads-ml.onrender.com
     - ADMIN_EMAIL: gabriel@unclicmarketplace.com.br
     - ADMIN_PASSWORD: (crie uma senha forte)
     - EDUZZ_WEBHOOK_SECRET: (deixe em branco por enquanto)
  5. Clique "Deploy" e aguarde 2-3 minutos

PASSO 3: Eduzz (15 minutos)
  1. Acesse https://integrations.eduzz.com/webhook/configs
  2. Crie nova chave de acesso (copie o valor)
  3. Volte para Render e preencha EDUZZ_WEBHOOK_SECRET
  4. Configure webhook na Eduzz apontando para:
     https://dashboard-ads-ml.onrender.com/webhook/eduzz
  5. Configure email pós-compra com instruções de cadastro

================================================================================
SUPORTE RÁPIDO
================================================================================

Erro 502 Bad Gateway?
→ Espere 2 minutos e recarregue. Verifique logs no Render.

Webhook retorna 401?
→ Verifique se EDUZZ_WEBHOOK_SECRET está correto.
→ Espere 1 minuto após salvar no Render.

Não consigo logar?
→ Verifique no painel admin se usuário existe com status "active".

Esqueci a senha de admin?
→ No Render, defina ADMIN_PASSWORD novamente e salve.

================================================================================
ARQUITETURA
================================================================================

[Cliente] → HTTPS → [App Python (Render)] → SQLite (disco persistente)
                          ↑
                          │ (webhook)
                          │
                    [Eduzz Orbita]

Fluxo:
1. Cliente compra na Eduzz
2. Eduzz envia webhook para /webhook/eduzz
3. App cria usuário com status "active"
4. Cliente acessa /cadastrar e define senha
5. Cliente faz login em /login
6. Sessão única: se logar em outro lugar, primeira sessão cai
7. Painel admin em /admin para gerenciar usuários
